# Hot ledger: one file per event

The ledger is where events live — records of "someone stored this kino with this content at this time." Each event goes into its own file, named by the hash of the event's canonical JSON encoding:

```
.kinora/hot/<ab>/<event-hash>.jsonl
```

One line, one event, immutable. The file is written once via tmp+rename; nothing ever mutates it after that.

Why one file per event? Because it makes the ledger **set-union mergeable**. Two git branches that independently record events produce disjoint filenames — the merge is "take the union of files." No JSONL is ever concatenated at merge time; no branch-name encoded into any filename; no conflicts from the ledger layer.

Files are sharded on the first two hex characters (matching the content store), so a shard directory stays browsable even with thousands of events.

Idempotency falls out for free: storing "the same logical event" twice produces the same event hash, hence the same filename. The second write is a no-op — we see the file exists and return early.
