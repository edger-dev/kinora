# Identity and versions

A kino has two kinds of identity that must not be conflated:

- **Identity** — the `id` field. Set once, at birth, equal to the content hash of the first version. Stable forever. Comparing two kinos for "is this the same thing?" means comparing ids.
- **Version** — the `hash` field. The content hash of a specific revision. Every edit produces a new version with a new hash; the id stays constant.

Versions link to each other through `parents`. A birth event has no parents (`parents == []`) and `id == hash`. A version event must declare its parents — the hashes it was derived from. The DAG this forms answers:

- "What is the current version?" → events under a given id whose hash nobody else lists as a parent. The *heads*.
- "Did someone fork?" → more than one head.
- "Did someone merge?" → a head with more than one parent.

There is no HEAD pointer that needs to be updated on every write. Current state is derived from the ledger by looking at the shape of the DAG. This keeps the ledger coherent under concurrent writes from independent agents — there is no shared pointer to contend over.
